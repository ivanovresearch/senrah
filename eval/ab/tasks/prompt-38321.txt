ISSUE TITLE: InvalidOperationException while filtering by property mapped to json column in db view in code first approach

ISSUE BODY:
### Bug description

Newest version of `Microsoft.EntityFrameworkCore.SqlServer` throws `InvalidOperationException` while filtering by property mapped to json column in db view in code first approach.

Same code works fine in  `Microsoft.EntityFrameworkCore.SqlServer` 9.0.16.

Example database contains `PersonOrdersView` dbview with `OrderIds` json column.
[Link to code in example repo](https://github.com/Kolan92/SimpleJsonColumnInView/blob/b8685beab0cf1ac4dc84fa1e647aed5a508cd6df/SimpleJsonColumnInView/InitializeDb.sql#L28)
```sql
CREATE OR ALTER VIEW PersonOrdersView
AS
SELECT 
PersonID,
LastName,
FirstName,
(
	SELECT DISTINCT
	Orders.OrderID as Value
	FROM Orders 
	WHERE  Orders.PersonID = Persons.PersonID FOR JSON PATH
) as OrderIds
FROM Persons
```

This view is mapped in following way DbContext while using database first approach:
[Link to code in example repo](https://github.com/Kolan92/SimpleJsonColumnInView/blob/b8685beab0cf1ac4dc84fa1e647aed5a508cd6df/SimpleJsonColumnInView/SimpleDbContext.cs#L24)
```csharp
modelBuilder.Entity<PersonOrdersView>(entity =>
{
    entity.ToView("PersonOrdersView");
    entity.OwnsMany(e => e.OrderIds, builder => builder.ToJson());
});
```

Type of `OrderIds` property is `List<ValueJson>` and `ValueJson` class is decorated with `[ComplexType]` attribute. 
[Link to code in example repo](https://github.com/Kolan92/SimpleJsonColumnInView/blob/b8685beab0cf1ac4dc84fa1e647aed5a508cd6df/SimpleJsonColumnInView/Models/ValueJson.cs#L5)

EF fails to generate a sql query while executing code with filtering clause on json collection.
```csharp
Random random = new();
var r = random.Next(1, 10);
// Reading collection of db views with json columns with filtering works fine in EF core 9.0.16, but throws in 10.0.8
var dbCollectionFiltered = await dbContext.PersonOrdersViews.Where(po => po.OrderIds.Any(oId => oId.Value == r)).ToListAsync();
```
[Link to code in example repo](https://github.com/Kolan92/SimpleJsonColumnInView/blob/b8685beab0cf1ac4dc84fa1e647aed5a508cd6df/SimpleJsonColumnInView/Controllers/AppController.cs#L28)

Interestingly ef core generates proper sql and executes it, when filtering is skipped:
```csharp
var dbCollection = await dbContext.PersonOrdersViews.ToListAsync();
```
